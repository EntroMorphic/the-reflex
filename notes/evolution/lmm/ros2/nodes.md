# Nodes: ROS2 Integration

## Node 1: The Elephant
ROS2 is ubiquitous. Incompatibility = irrelevance.
**Strategic:** Must integrate, not compete.

## Node 2: The Paradox
ROS2 is the problem AND the ecosystem we need.
**Resolution:** Bypass for control, use for everything else.

## Node 3: Complementary Positioning
"Complements ROS2, not replaces it."
**Why it matters:** Community politics.

## Node 4: What ROS2 Does Well
Coordination, messaging, ecosystem, community, abstraction.
**Acknowledge:** Give credit where due.

## Node 5: What ROS2 Does Poorly
Real-time, deterministic latency, high-frequency control.
**Opportunity:** Fill the gap.

## Node 6: The Bridge Package
ReflexChannelPublisher, ReflexChannelSubscriber, TelemetryNode.
**Purpose:** Connect the two worlds.

## Node 7: Shared Memory Architecture
Channels in mmap. ROS2 via wrappers. Reflex direct.
**Why it matters:** Zero-copy data transfer.

## Node 8: The Force Gripper Example
ROS2 sends setpoint → Reflex does 10kHz control → Anomaly alerts back to ROS2.
**Why it matters:** Concrete, understandable.

## Node 9: Distro Support
Humble (LTS) first. Iron, Jazzy later.
**Tension:** Supporting multiple distros is work.

## Node 10: micro-ROS
Embedded ROS2. Different architecture. Different integration.
**Question:** Do we support it? Priority?

## Node 11: Community Relations
Position as ally, not threat. Contribute upstream.
**Strategy:** Be a good citizen.
