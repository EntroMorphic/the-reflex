# Nodes: Support & Maintenance

## Node 1: Support = Survival
Product gets the sale. Support gets the renewal.
**Why it matters:** Retention > acquisition cost.

## Node 2: Three Tiers
Community (free), Standard (4hr), Premium (1hr 24/7).
**Mapping:** Open Core, Commercial, Premium.

## Node 3: Escalation Path
Customer Ops → Customer Lead → EntroMorphic Support → EntroMorphic Engineering.
**Goal:** Resolve at lowest level possible.

## Node 4: Maintenance Cadence
Daily telemetry, weekly health, monthly review, quarterly reforging.
**Principle:** Proactive > reactive.

## Node 5: Common Issues
Latency spikes (RT config), false positives (thresholds), detection misses (drift), integration failures (setup).
**Documentation:** Known issues, known solutions.

## Node 6: Reforging as Maintenance
Not failure. Maintenance. Instincts need updating.
**Normalize:** Set expectations upfront.

## Node 7: Documentation Investment
Good docs = fewer tickets. Invest heavily.
**Types:** Quick start, troubleshooting, architecture, API.

## Node 8: Support as Product
Excellent support is a differentiator. Not a cost center.
**Opportunity:** Upsell, cross-sell, referrals.

## Node 9: Scaling Concerns
Growing faster than support can scale = disaster.
**Mitigation:** Hire ahead, automate, self-service.

## Node 10: SLA Definition
What's critical? What's the response commitment?
**Clarity:** Set expectations, meet them.

## Node 11: Global Support
Different time zones. Different languages.
**Options:** Follow-the-sun, regional teams, async.

## Node 12: Customer Success vs Support
Support: fix problems. Success: ensure value.
**Both needed:** Reactive + proactive.
