# Nodes: Team Requirements

## Node 1: The Domain Expert Constraint
Irreplaceable. Can't hire. Has tacit knowledge.
**Why it matters:** No expert = garbage instincts.

## Node 2: Tacit Knowledge Extraction
Domain expert must articulate what they've never articulated.
**Challenge:** "The motor sounds different before it fails."

## Node 3: Minimum Viable Team
Robotics Engineer + Domain Expert + Project Lead. 3 people, part-time.
**Why it matters:** Accessible to most teams.

## Node 4: Production Team Expansion
+ Reflex Lead, Safety Engineer, more robotics engineers.
**Trigger:** Moving from pilot to production.

## Node 5: Skill Gap: Real-Time Linux
isolcpus, PREEMPT_RT, SCHED_FIFO. Very few people know this.
**Implication:** Training or hiring required.

## Node 6: Skill Gap: Low-Level C
Modern developers are too high-level. Need bare-metal skills.
**Implication:** May need embedded specialists.

## Node 7: Skill Gap: Hardware Interfaces
Memory-mapped I/O, DMA, direct peripheral access.
**Implication:** Hardware-software boundary expertise.

## Node 8: Gap-Filling Options
Train existing (slow, builds capability) vs co-develop (fast, creates dependency) vs hire (expensive, competitive market).
**Trade-offs:** Speed, cost, long-term capability.

## Node 9: Training Programs
Should EntroMorphic offer RT Linux training?
**Opportunity:** Revenue + qualified customers.

## Node 10: The Knowledge Transfer Goal
Customer should be self-sufficient eventually.
**Principle:** Build their capability, not their dependency.

## Node 11: Team Scaling Triggers
When to add people? What signals capacity limits?
**Indicators:** Deployment delays, quality issues, burnout.
