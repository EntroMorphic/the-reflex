# Raw Thoughts: Sales & Deployment Options

## Stream of Consciousness

Three tiers feels right. Captures the spectrum from hobbyist to enterprise.

OPEN CORE (Free)
The foot in the door. MIT license. Anyone can use it.
- Core primitive (reflex.h)
- Basic Reflexor
- Reference implementations
- Community support

Why give this away? 
1. Ecosystem building - more users = more mindshare
2. Prior art - harder for others to patent the basics
3. Developer love - open source builds trust
4. Pipeline - free users become paid users

COMMERCIAL (Subscription)
The product. Where the money comes from.
- Self-organizing Reflexors (the real magic)
- Forge SDK (train your own)
- Multi-Reflexor coordination
- Enterprise support with SLA
- Telemetry dashboards

Pricing: $2-10K/month based on fleet size. This feels right for robotics companies. A robot costs $50-500K. Our software is a rounding error if it works.

PREMIUM (Custom)
The moat. Safety-critical certifications + custom forging.
- ISO 13482 (service robots)
- ISO 10218 (industrial robots)  
- IEC 62443 (security)
- Custom Reflexor forging
- On-site deployment

Pricing: $50K-500K+ per engagement. High-touch, high-margin. This is where certification creates the moat - competitors can clone our tech but not our paperwork.

The quick start matters. Can someone go from zero to working demo in 30 minutes? If not, we lose them.

```bash
git clone ... && cd ... && make && sudo ./build/control_loop
```

Four commands. That's the target.

## Questions Arising

- How do we handle the free-to-paid conversion?
- What triggers an upgrade from Commercial to Premium?
- How do we price international vs domestic?

## First Instincts

Don't discount. Price reflects value. Discounting trains customers to expect discounts.

## What Scares Me

Racing to the bottom. If we compete on price, we lose. Compete on capability and trust.
