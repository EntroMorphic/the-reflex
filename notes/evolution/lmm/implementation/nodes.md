# Nodes: Implementation

## Node 1: Four Phases
Discover → Integrate → Forge → Operate.
**Why it matters:** Clear structure prevents chaos.

## Node 2: Discovery Importance
Don't skip. Every failed project skipped discovery.
**Activities:** Readiness, use case, success criteria, baseline.

## Node 3: Integration Foundation
RT kernel, core isolation, channels, bridge.
**Principle:** Shaky foundation = everything fails.

## Node 4: The Forge Process
Immersion → Observation → Crystallization → Validation.
**Duration:** 3-4 weeks for good instincts.

## Node 5: Immersion Week
Deploy unfrozen. Feed live data. Learn normal.
**Goal:** Reflexor absorbs environment patterns.

## Node 6: Observation Week
Delta Observer watches. Look for scaffolding. Collect validation data.
**Goal:** Know when learning is progressing.

## Node 7: Crystallization Week
Scaffolding dissolves = freeze. Validate R². Export Reflexor.
**Goal:** Capture learning at the right moment.

## Node 8: Validation Week
Test against known anomalies. Tune thresholds. Get approval.
**Goal:** Prove it works before production.

## Node 9: Operate Phase
Daily telemetry, weekly health, monthly review, quarterly reforging.
**Principle:** Production is the beginning, not the end.

## Node 10: Reforging Triggers
Accuracy drops, FP increases, environment changes, new anomalies.
**Why it matters:** Instincts need maintenance.

## Node 11: Timeline Pressure
"Can't we go faster?" No. Faster = failures = injuries.
**Principle:** Safety over speed.

## Node 12: Recovery Procedures
What if a phase fails? How do we recover?
**Answer:** Loop back, don't force forward.
