    PRD: Ternary Vector Database on LP Core                                                                                                                                                                                                        
                                                                                                                                                                                                                                                    
     Context                                                                                                                                                                                                                                        
                                                                                                                                                                                                                                                    
     We have a brute-force ternary VDB running on the LP core (16MHz RISC-V, ~30uA). It searches 16 nodes of 48 trits each, returns a single best match, and is verified exact against CPU reference. The LCVDB reference implementation (AArch64   
     HNSW in assembly) gives us a battle-tested architecture to port.                                                                                                                                                                               
                                                                                                                                                                                                                                                    
     Design Constraints                                                                                                                                                                                                                             
                                                                                                                                                                                                                                                    
     | Constraint | Value | Rationale |                                                                                                                                                                                                             
     |---|---|---|                                                                                                                                                                                                                                  
     | LP SRAM total | 16,320 bytes | Kconfig max feasible (HP needs ~36B of .rtc.data) |                                                                                                                                                           
     | Code + rodata | ~4,800 bytes | Current .text + .rodata + vector table |                                                                                                                                                                      
     | CfC state | ~968 bytes | Weights, hidden, dots, sync vars |                                                                                                                                                                                  
     | Stack | ~2KB minimum | CfC uses 96B frame, VDB will need ~128-256B |                                                                                                                                                                         
     | Available for VDB | ~8,000 bytes | SRAM - code - rodata - CfC - stack |                                                                                                                                                                      
     | Max nodes | 64 | 64 × node_size bytes |                                                                                                                                                                                                      
     | Vector dimension | 48 trits | Matches LCVDB's 48 dimensions |                                                                                                                                                                                
     | Trit packing | 16 trits/word | 3 words pos + 3 words neg = 24B per vector |                                                                                                                                                                  
     | Node IDs | uint8_t (6-bit) | 0..63, 0xFF = invalid |                                                                                                                                                                                         
     | M (max neighbors) | 4 | Layer 0 neighbor count |                                                                                                                                                                                             
     | M_upper | 2 | Upper layer neighbor count (if multi-layer) |                                                                                                                                                                                  
     | Wake period | 10ms | LP timer, ~100 Hz |                                                                                                                                                                                                     
                                                                                                                                                                                                                                                    
     Node Layout                                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                    
     The key design decision. LCVDB uses 64-byte cache-line-aligned nodes. We don't have a cache, but we need compact nodes to fit 64 in ~8KB.                                                                                                      
                                                                                                                                                                                                                                                    
     Option A: 32-byte nodes (tight)                                                                                                                                                                                                                
     [0..11]   pos_mask[3]    12 bytes (48-trit vector, positive)                                                                                                                                                                                   
     [12..23]  neg_mask[3]    12 bytes (48-trit vector, negative)                                                                                                                                                                                   
     [24..27]  neighbors[4]    4 bytes (4 × uint8_t neighbor IDs)                                                                                                                                                                                   
     [28]      neighbor_count   1 byte                                                                                                                                                                                                              
     [29]      layer            1 byte                                                                                                                                                                                                              
     [30..31]  reserved         2 bytes                                                                                                                                                                                                             
     64 nodes × 32 bytes = 2,048 bytes. Leaves ~6KB free. 4-word aligned.                                                                                                                                                                           
                                                                                                                                                                                                                                                    
     Option B: 24-byte nodes (current, no graph)                                                                                                                                                                                                    
     [0..11]   pos_mask[3]    12 bytes                                                                                                                                                                                                              
     [12..23]  neg_mask[3]    12 bytes                                                                                                                                                                                                              
     64 nodes × 24 bytes = 1,536 bytes. No room for neighbors.                                                                                                                                                                                      
                                                                                                                                                                                                                                                    
     Decision: Option A (32-byte nodes). The extra 8 bytes per node (512 bytes total for 64 nodes) buys us HNSW capability. M=4 neighbors is sufficient for 64 nodes — LCVDB uses M=8 for 256 nodes, so M=4 at 1/4 the scale is proportional.       
                                                                                                                                                                                                                                                    
     Memory Budget (with 32-byte nodes)                                                                                                                                                                                                             
                                                                                                                                                                                                                                                    
     | Section | Size | Notes |                                                                                                                                                                                                                     
     |---|---|---|                                                                                                                                                                                                                                  
     | Vector table | 128 | Fixed |                                                                                                                                                                                                                 
     | .text (code) | ~5,500 | Will grow ~700B with HNSW routines |                                                                                                                                                                                 
     | .rodata (popcount LUT) | 256 | Fixed |                                                                                                                                                                                                       
     | CfC BSS | 968 | Weights, hidden, dots, sync |                                                                                                                                                                                                
     | VDB nodes | 2,048 | 64 × 32 bytes |                                                                                                                                                                                                          
     | VDB metadata | 16 | node_count, entry_point, max_level, prng_state |                                                                                                                                                                         
     | VDB query + results | 56 | query (24B) + result_ids (4B) + result_scores (16B) + search_count (4B) + vdb_command (4B) |                                                                                                                      
     | shared_mem | 16 | At top of SRAM |                                                                                                                                                                                                           
     | Total static | ~9,000 | |                                                                                                                                                                                                                    
     | Stack | ~7,300 | 16,320 - 9,000. More than enough. |       
    Milestones                                                                                                                                                                                                                                     
                                                                                                                                                                                                                                                    
     ---                                                                                                                                                                                                                                            
                                                                                                                                                                                                                                                    
     M1: Top-K Results                                                                                                                                                                                                                              
                                                                                                                                                                                                                                                    
     Goal: Return the K best matches (K=4) instead of just the best one.                                                                                                                                                                            
                                                                                                                                                                                                                                                    
     Assembly changes (main.S):                                                                                                                                                                                                                     
     - In VDB_SCAN_LOOP: maintain a 4-element sorted list (descending by score) during brute-force scan                                                                                                                                             
     - After each INTERSECT_LOOP, insertion-sort the result into the top-K list                                                                                                                                                                     
     - Sorted list lives in callee-saved registers or stack: best_scores[4] (4 × int32 = 16B on stack) and best_ids[4] (4 × uint8 = 4B on stack)                                                                                                    
     - Algorithm: compare new score against best_scores[3] (worst of K). If better, overwrite slot 3 and bubble up (swap with slot 2 if better, then slot 1, then slot 0). Max 3 comparisons + swaps per insert.                                    
                                                                                                                                                                                                                                                    
     HP-side changes (geometry_cfc_freerun.c):                                                                                                                                                                                                      
     - Test 5a: self-match still returns id=7 at position 0                                                                                                                                                                                         
     - Test 5b: verify top-4 results against CPU brute-force top-4                                                                                                                                                                                  
     - New test: verify ordering (result0.score >= result1.score >= ...)                                                                                                                                                                            
                                                                                                                                                                                                                                                    
     Register plan for K=4 sorted insert:                                                                                                                                                                                                           
     Stack frame:                                                                                                                                                                                                                                   
       [sp+0..15]   best_scores[4]  (4 × int32)                                                                                                                                                                                                     
       [sp+16..19]  best_ids[4]     (4 × uint8)                                                                                                                                                                                                     
                                                                                                                                                                                                                                                    
     After INTERSECT_LOOP (result in s4, current node index in s0):                                                                                                                                                                                 
       lw  t0, 12(sp)       # worst score = best_scores[3]                                                                                                                                                                                          
       ble s4, t0, .Lnot_top4                                                                                                                                                                                                                       
       sw  s4, 12(sp)        # replace worst score                                                                                                                                                                                                  
       sb  s0, 19(sp)        # replace worst id                                                                                                                                                                                                     
       # bubble up: compare [3] vs [2], swap if better, then [2] vs [1], [1] vs [0]                                                                                                                                                                 
       ...                                                                                                                                                                                                                                          
     .Lnot_top4:                                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                    
     Estimated code growth: ~60 instructions = ~240 bytes.                                                                                                                                                                                          
                                                                                                                                                                                                                                                    
     Verification: K=4 results must exactly match CPU top-4 (same IDs, same scores, same order).

     M2: Scale to 64 + Benchmark                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                    
     Goal: Verify VDB works at full capacity (64 nodes) and measure latency.                                                                                                                                                                        
                                                                                                                                                                                                                                                    
     Changes:                                                                                                                                                                                                                                       
     - Test inserts 64 random ternary vectors (instead of 16)                                                                                                                                                                                       
     - Verify self-match and random-query at N=64                                                                                                                                                                                                   
     - Measure LP core search latency: set lp_command=2, timestamp before and after vdb_search_count increments                                                                                                                                     
     - Add a cycle counter or timestamp field in LP SRAM for the LP core to report its own timing                                                                                                                                                   
                                                                                                                                                                                                                                                    
     Timing estimation:                                                                                                                                                                                                                             
     - INTERSECT_LOOP: ~100 instructions per node (3 words × 4 ANDs × 4 LUT lookups = 48 LUT + 12 AND + 12 ADD + overhead)                                                                                                                          
     - At 16MHz, 100 instructions ≈ 6.25μs per node                                                                                                                                                                                                 
     - 64 nodes ≈ 400μs                                                                                                                                                                                                                             
     - With top-K overhead: ~450μs                                                                                                                                                                                                                  
     - Well within 10ms wake period. LP core can do CfC (~200μs) + VDB (450μs) = 650μs, then sleep for 9.35ms.                                                                                                                                      
                                                                                                                                                                                                                                                    
     Node layout change: Expand from 24-byte to 32-byte nodes in this milestone, even before HNSW. The neighbor fields will be zero but the memory is allocated. This validates the memory budget.                                                  
                                                                                                                                                                                                                                                    
     New BSS layout in main.S:                                                                                                                                                                                                                      
     VDB_NODE_BYTES = 32    # was 24                                                                                                                                                                                                                
     vdb_nodes: .space (64 * 32)  # 2,048 bytes (was 1,536)                                                                                                                                                                                         
                                                                                                                                                                                                                                                    
     Verification:                                                                                                                                                                                                                                  
     - 64/64 nodes inserted and searchable                                                                                                                                                                                                          
     - Self-match exact for all 64 nodes (loop test)                                                                                                                                                                                                
     - Random query matches CPU reference                                                                                                                                                                                                           
     - Search latency < 1ms reported

     M3: HP-Side API                                                                                                                                                                                                                                
                                                                                                                                                                                                                                                    
     Goal: Clean C API for the HP core to interact with the VDB.                                                                                                                                                                                    
                                                                                                                                                                                                                                                    
     New functions in geometry_cfc_freerun.c (or a new vdb_api.c):                                                                                                                                                                                  
                                                                                                                                                                                                                                                    
     /* Pack a trit vector and insert into VDB */                                                                                                                                                                                                   
     int vdb_insert(const int8_t *trit_vec_48);                                                                                                                                                                                                     
                                                                                                                                                                                                                                                    
     /* Search VDB for top-K matches */                                                                                                                                                                                                             
     int vdb_search(const int8_t *query_48, int k,                                                                                                                                                                                                  
                    uint8_t *result_ids, int32_t *result_scores);                                                                                                                                                                                   
                                                                                                                                                                                                                                                    
     /* Clear all nodes */                                                                                                                                                                                                                          
     void vdb_clear(void);                                                                                                                                                                                                                          
                                                                                                                                                                                                                                                    
     /* Get node count */                                                                                                                                                                                                                           
     int vdb_count(void);                                                                                                                                                                                                                           
                                                                                                                                                                                                                                                    
     vdb_insert just packs trits and writes to LP SRAM (no LP core involvement — insertion is HP-side SRAM writes). vdb_search packs query, sets lp_command=2, waits for completion, reads results.                                                 
                                                                                                                                                                                                                                                    
     For HNSW (M4): Insert will need to trigger the LP core for graph construction (find neighbors via dot products). New command: lp_command=3 (insert with neighbor selection). Or: keep brute-force insert on HP side and only do graph-aware    
     search on LP. Decision deferred to M4.                                                                                                                                                                                                         
                                                                                                                                                                                                                                                    
     Verification: All existing tests rewritten using the API. Cleaner, more readable test code.

     M4: HNSW Graph                                                                                                                                                                                                                                 
                                                                                                                                                                                                                                                    
     Goal: Graph-based approximate nearest neighbor search. O(log N) instead of O(N).                                                                                                                                                               
                                                                                                                                                                                                                                                    
     This is the big one. Adapted from LCVDB but simplified for our constraints.                                                                                                                                                                    
                                                                                                                                                                                                                                                    
     4a: Node Layout (32 bytes, finalized)                                                                                                                                                                                                          
     [0..11]   pos_mask[3]    vector positive                                                                                                                                                                                                       
     [12..23]  neg_mask[3]    vector negative                                                                                                                                                                                                       
     [24..27]  neighbors[4]   4 × uint8 neighbor IDs                                                                                                                                                                                                
     [28]      neighbor_count  uint8                                                                                                                                                                                                                
     [29]      layer           uint8                                                                                                                                                                                                                
     [30..31]  reserved        2 bytes (alignment)                                                                                                                                                                                                  
                                                                                                                                                                                                                                                    
     4b: VDB Metadata (16 bytes)                                                                                                                                                                                                                    
     vdb_node_count:   .space 4                                                                                                                                                                                                                     
     vdb_entry_point:  .space 1    # NEW                                                                                                                                                                                                            
     vdb_max_level:    .space 1    # NEW                                                                                                                                                                                                            
     vdb_prng_state:   .space 4    # NEW (xorshift32 for layer assignment)                                                                                                                                                                          
     vdb_pad:          .space 6                                                                                                                                                                                                                     
                                                                                                                                                                                                                                                    
     4c: Insert Algorithm (LP core assembly)                                                                                                                                                                                                        
                                                                                                                                                                                                                                                    
     Command lp_command=3: insert node at index vdb_node_count, vector already packed in the node slot by HP core.                                                                                                                                  
                                                                                                                                                                                                                                                    
     1. Layer assignment: xorshift32 PRNG, test 2-bit groups (p=1/4 per additional layer, since M=4). Layer 0: 75%, Layer 1: 18.75%, Layer 2: ~6%.                                                                                                  
                                                                                                                                                                                                                                                    
     2. Greedy descent from entry point to assigned layer (follow edges, compute dots against new vector, greedily advance to closer nodes). For N≤64 and 1-2 layers, this is 1-3 hops.                                                             
                                                                                                                                                                                                                                                    
     3. Candidate collection: Brute-force scan all nodes (N≤64, so brute is fine — same as LCVDB does for N≤256). Collect top-8 candidates (2×M).                                                                                                   
                                                                                                                                                                                                                                                    
     4. Diversity selection: From top-8, select M=4 neighbors using LCVDB's extended heuristic:                                                                                                                                                     
        - Sort candidates by score (descending)                                                                                                                                                                                                     
        - For each candidate: accept if it's not closer to an already-selected neighbor than to the query. Reject redundant edges.                                                                                                                  
                                                                                                                                                                                                                                                    
     5. Bidirectional edges: Write neighbors into new node. For each selected neighbor, add reverse edge (or replace weakest existing edge if full).                                                                                                
                                                                                                                                                                                                                                                    
     6. Update entry point if new node's layer > max_level.                                                                                                                                                                                         
                                                                                                                                                                                                                                                    
     Code estimate: ~400 instructions = 1,600 bytes. Total .text grows to 6,100 bytes. Still fits.                                                                                                                                                  
                                                                                                                                                                                                                                                    
     4d: Search Algorithm (LP core assembly)                                                                                                                                                                                                        
                                                                                                                                                                                                                                                    
     Command lp_command=2 (same as now, but graph-aware):                                                                                                                                                                                           
                                                                                                                                                                                                                                                    
     1. Greedy descent from entry point through upper layers.                                                                                                                                                                                       
     2. Beam search on layer 0: beam width = 4 (EF_SEARCH=4).                                                                                                                                                                                       
        - Visited bitset: 8 bytes (64 bits for 64 nodes) — fits in a register pair.                                                                                                                                                                 
        - Sorted beam: 4 scores + 4 IDs on stack (20 bytes).                                                                                                                                                                                        
        - Process each beam entry: for each neighbor, if unvisited, compute dot, insert into beam if it qualifies.                                                                                                                                  
     3. Return top-K from beam.                                                                                                                                                                                                                     
                                                                                                                                                                                                                                                    
     Fallback: If N < 8, fall back to brute-force (graph too sparse for meaningful search). This keeps the brute-force path alive.                                                                                                                  
                                                                                                                                                                                                                                                    
     Key simplification vs LCVDB: Visited bitset is only 8 bytes (64 bits) instead of 128 bytes. Fits in two registers (s-regs). No stack bitset needed.                                                                                            
                                                                                                                                                                                                                                                    
     Verification:                                                                                                                                                                                                                                  
     - Insert 64 nodes, search returns correct nearest neighbor                                                                                                                                                                                     
     - Graph connectivity: every node reachable from entry point                                                                                                                                                                                    
     - Search visits fewer nodes than brute-force (measure with a counter)                                                                                                                                                                          
     - Recall@1 >= 95% (exact match for top-1 vs brute-force ground truth)                                                                                                                                                                          
     - Recall@4 >= 90%

    M5: CfC → VDB Autonomous Pipeline                                                                                                                                                                                                              
                                                                                                                                                                                                                                                    
     Goal: The LP core wakes up, runs CfC on GIE hidden state, then immediately searches the VDB using the CfC output as a query — all in one wake cycle. Zero CPU.                                                                                 
                                                                                                                                                                                                                                                    
     New command: lp_command=4 (CfC + VDB combined step)                                                                                                                                                                                            
                                                                                                                                                                                                                                                    
     Assembly flow:                                                                                                                                                                                                                                 
     1. Read gie_hidden[32] + lp_hidden[16], pack to 48 trits                                                                                                                                                                                       
     2. Run CfC neuron loop (32 INTERSECT calls) → produce new lp_hidden[16]                                                                                                                                                                        
     3. Pack CfC output as VDB query: The concatenated [gie_hidden[32] | lp_hidden[16]] is already a 48-trit vector in packed form (it's the same x_pos[3]/x_neg[3] used as CfC input). Reuse it directly as the VDB query.                         
     4. Run VDB search (brute-force or HNSW)                                                                                                                                                                                                        
     5. Write results to vdb_result_ids / vdb_result_scores                                                                                                                                                                                         
     6. Also write CfC state as usual                                                                                                                                                                                                               
                                                                                                                                                                                                                                                    
     Timing:                                                                                                                                                                                                                                        
     - CfC: ~200μs                                                                                                                                                                                                                                  
     - VDB search (HNSW, 64 nodes): ~100-200μs (only visits ~10-15 nodes via graph)                                                                                                                                                                 
     - Total: ~300-400μs per wake cycle                                                                                                                                                                                                             
     - Duty cycle: 400μs / 10,000μs = 4%. LP core sleeps 96% of the time.                                                                                                                                                                           
                                                                                                                                                                                                                                                    
     What this means: The LP core autonomously:                                                                                                                                                                                                     
     - Perceives (reads GIE hidden state from peripheral neural network)                                                                                                                                                                            
     - Thinks (runs CfC liquid neural network)                                                                                                                                                                                                      
     - Remembers (searches VDB for similar past states)                                                                                                                                                                                             
                                                                                                                                                                                                                                                    
     All at ~30μA. No HP core involvement after init.                                                                                                                                                                                               
                                                                                                                                                                                                                                                    
     Demo scenario:                                                                                                                                                                                                                                 
     - Pre-load VDB with 32 "known state" vectors (e.g., from different GIE input patterns)                                                                                                                                                         
     - Run the combined pipeline                                                                                                                                                                                                                    
     - HP core periodically reads vdb_result_ids[0] — this is the "what am I experiencing right now?" answer                                                                                                                                        
     - Show the system classifying its own neural state in real-time, autonomously                                                                                                                                                                  
                                                                                                                                                                                                                                                    
     Verification:                                                                                                                                                                                                                                  
     - Combined CfC+VDB produces same CfC state as CfC-only                                                                                                                                                                                         
     - VDB results match what you'd get from separate CfC then search                                                                                                                                                                               
     - Pipeline runs at 100 Hz sustained (LP wakes every 10ms)                                                                                                                                                                                      
     - HP core reads results every 100ms, confirms updates
