# Reflect: Applications

## Core Insight

**Robotics is the throughline. Everything else is context.**

Aerospace = robots in space.
Medical = robots in surgery.
Industrial = robots in factories.
Defense = robots in conflict.
Automotive = robots on roads.

This unifies the market. One technology, one community, one sales motion. The "vertical" is just where the robot operates.

---

## Resolved Tensions

### Node 8 (Beachhead Selection)
**Resolution:** Start with **manipulation** (robot arms).

Why:
1. Largest market (millions of units)
2. Clear pain (force control, collision)
3. Fastest feedback loop (demo in minutes)
4. Universal need (every arm needs this)

Space is the best STORY but slowest SALES. Use space for vision, manipulation for revenue.

### Node 11 (Focus vs Breadth)
**Resolution:** Concentric circles.

Circle 1 (now): Manipulation (robot arms)
Circle 2 (6 months): Mobile + Drones
Circle 3 (12 months): Legged + Surgical
Circle 4 (18 months): Space

Each circle expands the market while leveraging what we've built.

## Cross-Application Patterns

All applications use the same primitives:
- Channel for communication
- Reflexor for anomaly detection
- Forge for training

Configuration changes:
- Different sensors
- Different actuators
- Different "normal" patterns
- Different response actions

**Build once, configure per application.**

---

## What I Now Understand

The application diversity is a strength, not a weakness. Same technology, different configurations. This creates:
1. Large total addressable market
2. Cross-learning between applications
3. Platform economics (build once, sell many)

The key is NOT trying to address all applications at once. Pick manipulation, dominate it, then expand. The concentric circles strategy prevents dilution while capturing optionality.

Space remains the vision even if it's not the beachhead. "Actual Instinct for robots - from factory floors to Mars" captures the full ambition.
